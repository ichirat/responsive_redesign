Responsive Redesign

Part 1: Identifying Usability Problems

Please access the original website here.

The website of choice is a E-Visa application portal. One could apply for E-Visa to Kyrgyzstan through this website.

Usability

  • The original website is not responsive. It is only designed to be acessed by PCs. Please try opening it on a phone or try adjusting the size of the browser on your PC.
  • The menu bar is not fixed at the top. Users have to scroll up to the top to be able to see it. Not a very efficient way to organize the website.
  • Links to the social media account of the website was placed at the bottom of the page without any emphasis and color contrast. They can be easily ignored.

  • The icons are not interactive. Users won’t be redirected if they just click the icons. Users have to click the text instead, contrary to most people’s first thought. However, the texts are neither underlined nor emphasized.

Learnability

  • The front image takes too much space without giving necessary information to the users. Indeed, users will know little about the website when they first open it.
  • The three main services provided by the website are not emphasized properly. Some users might be able to know what role do they play quickly.
  • The 16-second video is not that important. The “about us” section below it is more beneficial for the users to learn about the website.

Memorability

  • The website is rather simple without many advanced functions being designed, thus memorability is less of an issue for this specific website.
  • Yet, some of the issues in learnability and usability also apply here. For example, users can’t easily remember the links to the website’s social media platform (it’s hard to find and at the very bottom of the website).

Accessibility

Please view the report using this link if you wish.

I generally agree with the diagnosis. For example, it detects that the website has no heading or navigation bar. This is true from its source code. And this is also why it doesn’t have a fixed menu bar for users’ convenience. Also we could see the diagosis points out the color contrast is low

A few of the diagnosis does not make much sense to me, though. For example, the diagnosis says some of the icons do not have alternative text.

High-Fidelity Prototyping

Annotations (Please read)

  • I will develop the website using bootstrap.
  • All the elements will be made as a container within a flexbox. I will generally set margin using commands such as pt-5 or mb-5.
  • Fonts don’t need to be adjusted when users are in-between sizes. Indeed, the fonts won’t adjust to changes in screen size in my design. Users could play with font size somehow without interferring UX.
  • The page will change in response to user actions. Users could scroll up and down the page. All the buttons are designed to be interactive and will redirect the users to the actual visa application website (please don’t apply for the visa unless you need it).
